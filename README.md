### Installation

``` bash
$ npm install
```

or

``` bash
$ yarn install
```

### Basic usage

``` bash
# dev server with hot reload at http://localhost:3000
$ npm start 
```

or 

``` bash
# dev server with hot reload at http://localhost:3000
$ yarn start
```

Navigate to [http://localhost:3000](http://localhost:3000). The app will automatically reload if you change any of the source files.



## file structure

```
project-root/
│
├── src/                  # All application code goes here
│   ├── config/           # Configuration files
│   │   └── config.js     # Database configuration (and other services if needed)
│   │
│   ├── controllers/      # API logic to handle requests and responses
│   │   ├── authController.js
│   │   ├── employeeController.js
│   │   ├── timesheetController.js
│   │   ├── projectController.js
│   │   ├── teamController.js
│   │   └── reportController.js
│   │
│   ├── models/           # Sequelize models for the database tables
│   │   ├── user.js
│   │   ├── employee.js
│   │   ├── timesheet.js
│   │   ├── project.js
│   │   ├── team.js
│   │   ├── employeeTeam.js  # Many-to-many relationship model
│   │   └── projectTeam.js   # Many-to-many relationship model
│   │
│   ├── routes/           # Defines the API routes for each resource
│   │   ├── authRoutes.js
│   │   ├── employeeRoutes.js
│   │   ├── timesheetRoutes.js
│   │   ├── projectRoutes.js
│   │   ├── teamRoutes.js
│   │   └── reportRoutes.js
│   │
│   ├── middlewares/      # Middleware logic such as authentication and error handling
│   │   ├── authMiddleware.js
│   │   └── errorHandler.js
│   │
│   ├── migrations/       # Sequelize migrations (auto-generated by Sequelize CLI)
│   └── seeders/          # Sequelize seeders for populating initial data
│   
│
├── .env                  # Environment variables for sensitive info (e.g., DB credentials)
├── .gitignore            # Ignored files and folders (e.g., node_modules, .env)
├── server.js             # Main entry point of the application (app setup and start)
└── package.json          # Project metadata and dependencies

```
